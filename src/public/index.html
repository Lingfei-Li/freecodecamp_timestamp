Timestamp Microservice

User Stories:

  1. I can pass a string as a parameter, and it will check to see whether that string contains either a unix timestamp or a natural language date (example: January 1, 2016).
  2. If it does, it returns both the Unix timestamp and the natural language form of that date.
  3. If it does not contain a date or Unix timestamp, it returns null for those properties.

That being said, there are two functions in this service:

  1. Putting a unix timestamp (in seconds) in the route, and get the natural representation for it
    • Request: https://radiant-bayou-77456.herokuapp.com/1450137600
    • Response: {"natural":"2015-12-15","unix":1450137600}
  2. Putting a unix timestamp (in seconds) in the route, and get the natural representation
    • Request: https://radiant-bayou-77456.herokuapp.com/2015-12-15
    • Response: {"natural":"2015-12-15","unix":1450137600}
